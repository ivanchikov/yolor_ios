# UI/UX Review Checklist

## Product Flow

- [ ] Planning is reachable immediately on launch.
- [ ] Login is accessible from the global menu without blocking anonymous planning.
- [ ] Save still acts as the strongest contextual authentication prompt.
- [ ] Generation progress reflects real backend stages.

## Trip Review

- [ ] The selected day is obvious at a glance.
- [ ] Map and stop list stay synchronized when the day changes.
- [ ] Travel segments are visually distinct from stop rows.
- [ ] Edit actions are available without obscuring the day review.

## Editing

- [ ] Locked stops are visually obvious.
- [ ] Regeneration scope is explicit before the action runs.
- [ ] Failed edits restore the last valid state.
- [ ] User-added versus generated stops are distinguishable.

## Offline And Trust

- [ ] Saved trips are discoverable from the profile/menu path.
- [ ] Offline-readable data is clear in the saved-trips experience.
- [ ] Fallback behavior is explained in plain language.
- [ ] Missing provider data never appears as false certainty.
